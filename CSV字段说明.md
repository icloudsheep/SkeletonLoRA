# CSV 字段说明

本文件说明 SkeletonLoRA 新实验输出中的中文字段。所有 CSV 使用 `utf-8-sig` 编码，便于中文表格软件打开。

## 统计层级

实验指标同时保留以下层级：

| 层级 | 标识字段 | 含义 |
|---|---|---|
| 全局运行 | `运行 ID` | 一次独立时间戳 run |
| 配置 | `方法`、`实验模式`、`部分加密比例`、`骨架优化` | 一组可比较的 CKKS/明文配置 |
| 客户端 | `客户端 ID` | 单个客户端的加密、上传和解密统计 |
| AB 对 | `层名`、`投影名`、`AB 标识` | 一个独立 LoRA A/B 矩阵对 |
| 块 | `块类型`、`行起点`、`行终点`、`列起点`、`列终点` | partial/full 的明文或密文坐标块 |

## 配置字段

| 中文字段 | 类型/单位 | 说明 |
|---|---|---|
| `运行 ID` | 文本 | 时间戳命名的 run 目录名称 |
| `方法` | 文本 | `外积`、`内积` 或 `明文参考` |
| `实验模式` | 文本 | `plain_baseline`、`partial_A`、`partial_AB`、`full` |
| `部分加密比例` | 百分比 | `partial_A`/`partial_AB` 使用的比例；baseline/full 填 `N/A` |
| `骨架优化` | `开`/`关` | 是否只下发 skeleton 行列并使用 CUR |
| `客户端数量` | 整数 | 等权平均的固定分母 `N_CLIENTS` |
| `LoRA 秩` | 整数 | 全局 rank，默认 4 |
| `LoRA alpha` | 数值 | 全局 alpha，默认 8 |
| `LoRA scaling` | 数值 | `alpha / rank`，默认 2 |
| `skeleton rank` | 整数 | 配置的 skeleton rank，默认 16 |
| `实际 skeleton rank` | 整数 | `min(skeleton rank, N_CLIENTS × rank)` |
| `输出维度` | 整数 | B 的行数 |
| `输入维度` | 整数 | A 的列数 |
| `CKKS 多项式次数` | 整数 | `poly_modulus_degree` |
| `CKKS 槽位数` | 整数 | `poly_modulus_degree / 2` |

## 误差字段

| 中文字段 | 类型/单位 | 说明 |
|---|---|---|
| `相对 Frobenius 误差` | 无量纲 | `||结果-参考||F / ||参考||F` |
| `最大绝对误差` | 浮点数 | 所有元素绝对误差的最大值 |
| `平均绝对误差` | 浮点数 | 所有元素绝对误差的平均值 |
| `CKKS 直接结果误差` | 无量纲 | CUR 前解密矩阵相对参考的误差 |
| `骨架行列误差` | 无量纲 | 解密并拼接 skeleton 行列后的误差 |
| `CUR 重建误差` | 无量纲 | CUR 完整矩阵相对参考的误差 |
| `密文区域误差` | 无量纲 | partial 输出中密文坐标区域的误差 |
| `明文区域误差` | 无量纲 | partial 输出中明文坐标区域的误差 |
| `精度状态` | 文本 | `正常`、`warning` 或 `failure`；默认阈值为 `1e-3` 和 `1e-1` |

## 时间字段

所有时间字段单位为秒。主指标不将密钥生成时间混入 AB 对耗时。

| 中文字段 | 说明 |
|---|---|
| `context 创建秒` | 创建 secret context 的耗时 |
| `context 派生秒` | 从 secret context 派生 public context 的耗时 |
| `Galois key 生成秒` | 内积方法生成 Galois key 的耗时 |
| `客户端编码秒` | 明文向量编码准备时间 |
| `客户端加密秒` | CKKS 加密及客户端序列化时间；客户端级别另外记录平均值和最大值 |
| `服务端反序列化秒` | 服务端加载 CKKS 密文的耗时 |
| `服务端同态运算秒` | 同态乘法、加法和平均的耗时 |
| `服务端序列化秒` | 服务端结果序列化耗时 |
| `服务端聚合秒` | 服务端反序列化、同态运算、跨客户端平均和序列化总耗时 |
| `客户端反序列化秒` | 客户端加载服务端下发密文的耗时 |
| `客户端解密秒` | 客户端 CKKS 解密耗时 |
| `客户端拼接秒` | 明文块与解密密文块按坐标合并的耗时 |
| `CUR 秒` | 交叉块 SVD、求逆/伪逆和矩阵重建耗时 |
| `客户端总解密秒` | 反序列化、解密、拼接和 CUR 的总耗时 |
| `明文参考秒` | NumPy 明文参考计算耗时 |

## 通信字段

默认通信比较以 CKKS `serialize()` 返回的序列化字节数为准；落盘文件大小单独记录。

| 中文字段 | 类型/单位 | 说明 |
|---|---|---|
| `上传密文字节` | 字节 | 客户端上传 CKKS 序列化密文大小 |
| `上传明文字节` | 字节 | 客户端上传明文 NumPy payload 大小 |
| `上传元数据字节` | 字节 | AB 标识、形状、块类型和索引范围等协议元数据 |
| `单客户端上传字节` | 字节 | 上述三类上传字节之和 |
| `下发密文字节` | 字节 | 服务端下发 CKKS 序列化密文大小 |
| `下发明文字节` | 字节 | 服务端下发明文 payload 大小 |
| `下发元数据字节` | 字节 | 服务端下发结果的协议元数据 |
| `单次下发字节` | 字节 | 上述三类下发字节之和 |
| `聚合阶段总网络字节` | 字节 | N 个客户端上传加服务端一次下发 |
| `落盘文件字节` | 字节 | 启用二进制产物保存时的文件实际大小 |
| `初始化 context 字节` | 字节 | public context 的一次性序列化大小 |
| `生命周期总网络字节` | 字节 | 初始化 context 通信加聚合阶段总网络 |

## CUR 与可行性字段

| 中文字段 | 说明 |
|---|---|
| `交叉块数值秩` | `M` 的 NumPy 数值秩 |
| `最大奇异值` | `M` 奇异值最大值 |
| `最小奇异值` | `M` 奇异值最小值 |
| `交叉块条件数` | `cond(M)` |
| `逆方法` | `inv` 或 `pinv` |
| `可行性` | `可行`、`部分成功` 或 `不可行` |
| `成功 AB 数` | 当前配置成功完成的 AB 对数量 |
| `失败 AB 数` | 当前配置失败的 AB 对数量 |
| `失败比例` | 失败 AB 数除以总 AB 数 |
| `失败原因` | 结构、资源、CKKS、CUR 或其他异常说明 |

`不可行` 的数值字段使用 `N/A`，不使用 0；真正的零耗时或零通信才记录为 0。
